RStack is a governed AI-SDLC control plane: a 15-stage pipeline (00-environment â†’ 14-cost-estimation)
with builder/validator contracts, evidence ledgers, approval gates, enforced guardrails, and the
Business Hub observability dashboard (`npm run business`, port 3008). The host framework
(Pi / Claude Code / Operator) executes the agents; RStack owns run state, contracts, and governance.

## Product goals (north star)

1. **Governed loop, enforced in code** â€” every guardrail, gate, and contract promise in the docs is
   backed by runtime enforcement in `src/core/harness/`, never by prompt text alone.
2. **Client-ready end-user product** â€” a team can install `rstack-agents`, run `init`, and get a
   working governed pipeline with observable state, without reading the source.
3. **Brownfield first-class** â€” adopting an existing codebase (reverse-populating stages 00â€“06 from
   real artifacts) is as supported as greenfield. Tracked by epic #148.
4. **Transparent state** â€” pipeline state is authoritative, atomic, schema-versioned, and inspectable
   from CLI (`rstack-agents pipeline status`), dashboard, and JSON â€” never silently degraded.
