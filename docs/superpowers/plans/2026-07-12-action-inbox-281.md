# Normalized Action Inbox Implementation Plan

RStack developed by Richardson Gunde

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Normalize every human-attention source into one deterministic, source-linked Action Inbox used by Overview and Decisions.

**Architecture:** Add a pure state projection with per-source producers, canonical dedupe, and stable ordering. Serialize the bounded list, render one read-first Inbox, and replace Overview’s local attention count/action fallback with this shared queue.

**Tech Stack:** Node.js ES modules, vanilla JavaScript dashboard, CSS, Node test runner.

## Global Constraints

- No client-invented mutation or identity.
- Stale/invalid/unavailable sources fail closed.
- Approval lifecycle/audit data is preserved verbatim.
- #274 queue output is authoritative; task/pipeline derivation is a deduped safety net.
- Desktop and 390px keyboard access are required.
